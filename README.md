# gslb_rust

[![Python CI](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-python.yml/badge.svg)](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-python.yml) [![Node CI](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-node.yml/badge.svg)](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-node.yml) [![Browser CI](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-browser.yml/badge.svg)](https://github.com/gilzim/rust-gslb-library/actions/workflows/ci-browser.yml) [![Release](https://img.shields.io/github/v/release/gilzim/rust-gslb-library)](https://github.com/gilzim/rust-gslb-library/releases/latest)

A blazing‑fast, minimal‑footprint Global Server Load Balancing (GSLB) client library. Powered by a native Rust engine, this library provides bindings for **Python**, **Node.js**, and the **Browser** (via WebAssembly). The current release is **1.2.0**.

`gslb_rust` allows applications to perform client‑side load balancing with intelligent latency‑based routing, Weighted Round Robin (WRR), and instant failover. All of this is achieved without blocking the main execution thread of your host environment (such as the Python GIL or the Node.js/Browser event loop) and requires minimal CPU and memory overhead.

## 🚀 Features

* **Zero Overhead:** Background health checks and latency monitoring run in native Rust OS threads (or WASM workers).
* **Latency-Aware Routing:** Automatically routes traffic to the fastest responding endpoint.
* **Tolerance Margins:** Group nodes with similar latencies (e.g., within 30ms of each other) to prevent traffic flapping.
* **Weighted Round Robin (WRR):** Distribute traffic proportionally among optimal endpoints (e.g., 75% to US-East, 25% to US-West).
* **Instant Failover:** Manually report failures mid-flight to instantly strike a node from the routing pool.
* **No External Dependencies:** Statically linked core, requiring no system-level OpenSSL installations.

## 📦 Installation & Quick Start

The repository produces artifacts for three distinct environments, all generated by a unified CI workflow and attached to each GitHub release.

### Python

Pre‑built wheels for supported platforms and Python versions (CPython 3.9 through 3.12) are available on the release page.

```bash
pip install gslb_rust-<version>-<platform>.whl
# Or via PyPI if published: pip install gslb_rust
```

```python
import gslb_rust

nodes = ["https://us-east.api.com", "https://eu-west.api.com"]
resolver = gslb_rust.GslbResolver(nodes, interval_secs=5, latency_margin_ms=20)
resolver.spawn_monitor()

# Use in your loop - takes < 1 microsecond
best_endpoint = resolver.get_endpoint()
print(f"Routing to: {best_endpoint}")
```

### Node.js

The `js-node/` crate builds a NAPI native module packaged as an npm tarball (`.tgz`). Download the `gslb-node-*.tgz` artifact from a release and install it.

```bash
npm install /path/to/gslb-node-1.2.0.tgz
```

```javascript
const { GslbResolver } = require('gslb-node');

const nodes = ["https://us-east.api.com", "https://eu-west.api.com"];
// Checks every 5 seconds, 20ms latency margin
const resolver = new GslbResolver(nodes, 5, 20);
resolver.spawnMonitor();

const bestEndpoint = resolver.getEndpoint();
console.log(`Routing to: ${bestEndpoint}`);
```

### Browser / WebAssembly

The `browser-wasm` artifact directory contains a precompiled WebAssembly bundle and JavaScript glue code that can be dropped directly into a web app or bundler.

```html
<script type="module">
  import init, { GslbResolver } from './pkg/gslb_browser_source.js';
  await init();
  
  const nodes = ["https://us-east.api.com", "https://eu-west.api.com"];
  // Checks every 5 seconds, 20ms latency margin
  const resolver = new GslbResolver(nodes, 5, 20);
  resolver.spawn_monitor();
  
  const bestEndpoint = resolver.get_endpoint();
  console.log(`Routing to: ${bestEndpoint}`);
</script>
```

Alternatively, run one of the `wasm-pack` targets yourself from the `js-browser` directory.

## ⚙️ Advanced Usage

Advanced features are consistent across all language bindings. The concepts outlined below apply universally.

### Weighted Round Robin (WRR)

By default, each healthy endpoint contributes an equal share to the routing pool. When balancing among clusters with differing capacity, you can supply a *weight* for each URL. The resolver first filters nodes by latency margin and then applies WRR to the remaining set.

For example, assigning a weight of `3` to a fast node and `1` to a slow node means approximately 75% of calls will hit the fast endpoint. By default endpoints have an equal weight of `1`.

### The "Latency Margin" Explained

Strict latency routing can cause "flapping" (where 100% of traffic wildly swings between two servers that are only 1ms apart). 

The `latency_margin_ms` parameter fixes this. If your fastest node responds in 40ms, and your margin is set to 20, any healthy node that responds in **60ms or less** is added to the active routing pool. Traffic is then distributed among that pool using your configured WRR weights.

### Manual Circuit Breaker

Sometimes a server responds to a background health ping but fails during actual application logic. You can report this to instantly pull the node from rotation.

```python
target = resolver.get_endpoint()

# Request failed? Instantly pull the node from active rotation
resolver.report_failure(target)

# Fallback to the next best node immediately
new_target = resolver.get_endpoint()
```

### Health Reporting

Want to see current latencies without parsing logs? Use the built-in report generator.
```python
stats = resolver.get_report()
print(stats) # e.g., {'https://us-east.api.com': 42, 'https://eu-west.api.com': 115}
```

## 🛠️ Building from Source

To build locally, you'll need the Rust toolchain installed. Depending on the binding you want to build, you will also need Python, Node.js, and/or `wasm-pack`.

1. **Install Rust:**
   ```bash
   curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
   ```
   
### Building the Python Binding

Requires `maturin`.

```bash
# Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate
pip install maturin pytest

# Build the Python extension
maturin develop --release
pytest tests/test_gslb.py
```

### Building the Node.js Binding

Requires Node.js (>= 14) and `npm`.

```bash
cd js-node
npm install

# Build using napi-rs (as defined in package.json)
npm run build
```

### Building the Browser / WebAssembly Binding

Requires `wasm-pack`.

```bash
cargo install wasm-pack
cd js-browser
npm install

# Build WASM for web, bundler, or nodejs
npm run build
# The resulting wasm and js files will be generated in js-browser/pkg/
```
